GDS – GeoJSON API with JWT Security

Role: Backend & GIS Developer
Tech: Python, Flask, JWT, SQLAlchemy, GeoJSON
Focus: Secure APIs for geographic data

Overview

GDS is a service-oriented backend application for managing geographic data stored in GeoJSON format. It exposes a set of REST API endpoints that let clients create, read, update, and delete GeoJSON features, while keeping all access secure using JSON Web Tokens (JWT).

The project is structured to be modular and maintainable, with separate layers for models, routes, services, and utilities. This makes it easy to extend with new geographic operations or additional data sources over time.

Problem & Motivation

I wanted a clean, reusable way to interact with GeoJSON data through an API instead of manually opening files in a GIS tool every time. At the same time, I needed proper authentication so that only authorised users could access or edit sensitive geographic datasets.

Data & Domain

  • GeoJSON feature collections representing geospatial layers.
  • Typical content: boundaries, structures, or other vector layers.
  • Optional database backing (e.g. via SQLAlchemy) for persistence.

What the System Does

1. JWT Authentication

  • Every protected endpoint requires a valid JWT in the request headers.
  • Tokens are signed using a secret key defined in JWT_SECRET_KEY.
  • Only authenticated users can access API routes that expose or modify data.

2. GeoJSON Processing

  • Read and write GeoJSON files on disk.
  • Utility functions to manipulate features and properties.
  • Standardised format makes it easy to load the results in QGIS or other tools.

3. Modular Code Structure

  • models/: data models and ORM definitions.
  • routes/: API endpoint definitions.
  • services/: business logic and GeoJSON operations.
  • utils/: helpers for authentication, configuration, etc.

How I Implemented It

  1. Set up the backend stack using Python and Flask, plus virtual environment and dependency management.
  2. Designed the authentication flow: user login → JWT issued → token checked on every request.
  3. Implemented GeoJSON utilities to load and save feature collections and to validate the format.
  4. Organised the project into modules so that each part (auth, data handling, routing) has a clear responsibility.

Setup & Usage (Developer View)

The project is set up like a typical Python web service:

  • Clone the repo and create a virtual environment.
  • Install dependencies with pip install -r requirements.txt.
  • Set environment variables including JWT_SECRET_KEY and any database URLs.
  • Start the Flask app and interact with endpoints via HTTP clients or a frontend.

What I Learned

  • How to design a secure JWT-based authentication layer for an API.
  • How to structure a Python project into models, routes, services, and utilities.
  • Practical experience working with GeoJSON outside of desktop GIS tools.
